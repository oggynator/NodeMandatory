Running Node.js

How do we run Node.js ?

A Node.js project can be executed in different ways, depending on what the developer wish to achieve, this page will describe some different ways to run code.

Via the terminal 

node app.js

This simply runs the node project, no new changes will be detected and the developer has to restart the app to execute new changes


nodemon app.js

After adding the nodemon dependency, the developer can use the nodemon tool to continueasly and automatically restart the project when new changes are detected 


budo index.js --live --open

After installing budo (often via a npx command), the developer can use the tool to "browserify" the development. This means that the code is automatically rerun on new changes and the browser window automatically refreshes so the developer can instantenously see the changes.


Via the run script in the package.json file

	"scripts": {
		"test": "echo \"Error: no test specified\" && exit 1",
		"start": "cross-env PORT=80 node app.js",
		"start-dev": "cross-env PORT=3000 nodemon app.js"
	  },

The developer can define run scripts in a JSON format in the "package.json" file. By defining different scripts, the developer can run the web app with different conditions. One script might be used for running the web app in production, one might be used for testing and one might be used for development.

In the code above, we combine nodemon and the cross-env dependency and define a "start-dev" command that can be used for development. We also define different ports depending on how we wish to run the code. We execute the script by starting it from the terminal, like in the example below:

npm run start-dev
