NPM

What is NPM?

Node Package Manager or npm for short, is a free to use package manager. Node.js comes bundled with npm, so no additional installation is required. It was created specifically for Node.js, hence the name, but it can also be used for other projects.
npm enables the developer the reuse already developed software components, this speeds up the development process and provides well-tested functionality to the project.

How to use npm in a Node.js project

All npm dependencies is defined in a JSON file called “package.json”. A smart way to start define the “package.json” file, is to use the “npm init” command which will fill out the file with information such as package name and version number, but also a lot of other information. Added dependencies must be defined in the “dependencies” key value pair.

NPM commands are written in the terminal, some (but far from all) of the commands we have used so far are:

npm init

As explained above, this initalizes the "package.json" file which enables the developer to use npm.


npm install (PACKAGE NAME)

This installs a npm package, the dependency is automatically added to the dependency key value pair in "package.json" file. A new package can be installed either locally or globally, in general a local install is preferable. 


npm run (SCRIPT NAME)

This runs a given script from the "package.json" file, this gives the developer the option of choosen how they want to run their project.


Some of the NPM packages that we have used

Express: A small web development framework, it provides the developer with a set of useful features for web development.

Nodemon: A tool that will restart the application when changes in the source are detected, prevents the developer from manually having to restart the application.

cross-env: Enables the developer to use environment variables, without having to be concerned about which platform execute the program
